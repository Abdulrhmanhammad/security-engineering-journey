# CTF Write-ups

Every solved challenge gets a write-up — this is the clearest evidence of
applied skill in the whole repo: real target, real constraints, real solution.

## Structure

```
ctf-writeups/
  TEMPLATE.md            <- copy this for every new write-up
  tryhackme/
  hackthebox/
  picoctf/
  defcon-quals/
  other/
```

Create a platform folder the first time you need it (not all listed above may
apply — add more as needed, e.g. `ctfd/`, `local-ctf/`).

## Rules for a write-up in this repo

- Only challenges actually solved by hand — no copy-pasted public write-ups.
- Include the full path: recon → what was tried and failed → what worked → root cause.
- Redact/replace any flag values, private lab credentials, or VPN configs before committing.
